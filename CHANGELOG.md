# Changelog

All notable changes to this project will be documented in this file. The format loosely follows [Keep a Changelog](https://keepachangelog.com/en/1.1.0/).

## [Unreleased]

## [0.4.3] - 2026-05-21
- TG 频道用途改为「关闭 / 仅搜索 / 同步+搜索」三态，兼容旧启用状态，并区分同步展示频道与参与搜索频道。
- 资源页顶部源数量标签支持按状态切换：默认显示「同步源」，搜索结果显示「搜索源」，缓存数量保持本地 TG 缓存口径。
- 收短资源卡片导入动作按钮，分享资源显示「转存」、磁力资源显示「下载」，避免移动端按钮换行。

## [0.4.2] - 2026-05-20
- 重构资源推荐页的 Explore 筛选区：桌面端改为双栏筛选布局，移动端收敛为快筛条 + 底部抽屉，减少竖屏首屏拥挤和横屏断点冲突。
- 资源推荐列表卡片收敛为“标题一行 + 次信息一行”的紧凑结构，移除列表简介，统一同排卡片高度，并让长原名在次信息行尾部单行截断。

## [0.4.1] - 2026-05-19
- 修复资源中心筛选栏在窄屏幕下被压缩导致背景和文字显示异常的问题，添加 `flex-shrink: 0` 保持按钮组自然宽度。

## [0.4.0] - 2026-05-19
- 全局 API 认证中间件：所有接口（除登录、favicon、userscript）需登录后才能访问，未认证请求自动跳转登录页。
- Session Secret Key 改为首次启动自动生成随机值并持久化，不再硬编码。
- 登录密码改用 bcrypt 哈希存储和验证，替换明文比较。
- 新增 CSRF 防护：所有 POST 请求需携带 CSRF Token，登出改为 POST 方法。
- CORS 默认策略收紧：默认仅允许同源访问，需跨域时通过环境变量显式配置。
- 登录失败限流：连续失败 5 次后锁定 5 分钟，防止暴力破解。
- 修复数据库连接泄漏：所有 `open_db()` 调用统一使用 `try/finally` 确保连接关闭。
- 修复并发竞态：共享集合（`resource_job_running` 等）和队列（`monitor_queue` 等）添加锁保护。
- 修复 `ensure_db()` 双重检查锁定缺陷，DDL 操作移入锁内。
- 关键异常处理补充日志：`except Exception: pass` 改为记录错误详情，便于排查。
- 错误响应脱敏：`str(exc)` 不再直接返回客户端，改为返回通用错误信息。
- 阿里云盘、天翼云盘、TMDB API 调用增加重试机制。

## [0.3.29] - 2026-05-16
- 统一文件管理列表的表头与名称列对齐方式，修复资源目录、订阅保存目录、监控文件夹和刮削文件列表中的名称列居中或错位问题。
- STRM 残留目录清理的目录列表去掉多余外框，并把滚动交给列表本身，避免弹窗内表格出现嵌套边框和内容裁切。
- 刮削页选择状态改为只显示选择数量，识别模式和预览模式的长说明改为悬停提示，避免状态文案遮挡操作按钮。
- 刮削页竖屏操作区改为紧凑网格布局，并压缩按钮尺寸，减少预览模式下的换行和横向裁切。

## [0.3.28] - 2026-05-15
- STRM 生成链接改为只保留服务地址与远端文件路径，不再写入 `pickcode`，避免同一路径文件因辅助参数变化被反复判定为更新。
- 文件夹监控的 STRM 更新语义与播放链接保持一致：服务地址或路径变化才重写；同名同路径替换文件不会触发 STRM 文件更新。
- 按路径播放时默认刷新当前目录文件解析，并关闭 path 到 pickcode 的默认长缓存，减少同名替换后继续使用旧文件标识的风险。

## [0.3.27] - 2026-05-14
- 订阅任务新增“精准匹配”开关，开启后只认片名、分享目录、文件路径或 TMDB ID 这类身份线索，简介、主演和正文叙述里的命中不再算同剧证据。
- 订阅候选评分拆分普通召回文本与标题身份文本，避免《主角》这类泛化剧名只因出现在其他剧简介里就进入导入流程。
- 115 / 夸克分享清单和文件级选集增加 TMDB ID 冲突拦截，候选或文件路径带其他 `{tmdb-...}` 时会跳过并记录精准匹配过滤原因。

## [0.3.26] - 2026-05-14
- 订阅任务状态写入后会主动推送轻量 UI 更新，SSE 增量状态同步携带任务进度快照，避免日志和通知已完成但前端卡片仍停留在运行进度条。
- 资源导入任务创建、运行和完成时新增轻量任务状态信号，前端会先本地同步资源卡片状态再短延迟刷新，减少手动导入完成后卡片长时间显示“导入中”的情况。

## [0.3.25] - 2026-05-13
- 文件夹监控页顶部状态区改为固定栅格布局，长状态路径在状态框内单行省略，避免任务运行时挤压按钮和任务列表导致页面闪烁。
- 当前状态框补充完整状态的悬停提示和无障碍标签，省略显示时仍能查看完整任务状态。

## [0.3.24] - 2026-05-13

- 通知标题高亮会转义资源标题中的特殊字符，避免资源名里的 `<`、`>` 或 `&` 影响企业微信 Markdown 渲染。

## [0.3.23] - 2026-05-13

## [0.3.22] - 2026-05-13
- 更新版本号到 0.3.22，为首页资源卡片状态刷新优化预留新的发布点。

## [0.3.21] - 2026-05-12
- 新增后台内存 housekeeping，定期清理资源状态、STRM、图片代理、115、夸克和 TMDB 等运行时缓存，并在清理后尝试释放进程空闲内存。
- 资源搜索取消标记改为 TTL 自动过期，避免异常中断或漏清理的搜索 ID 长期留在内存中。
- 资源状态快照缓存增加过期清理和容量上限，减少多端刷新或分页状态变化带来的短期缓存堆积。
- STRM 下载地址、中继 token、pickcode 路径和 folder cid 缓存统一按 TTL 与容量剪枝，降低长时间播放/浏览后的常驻内存增长。
- 资源频道配置变更后会清理已删除频道的运行态缓存，避免 `last_sync`、错误信息和频道画像保留无效数据。

## [0.3.20] - 2026-05-12
- 优化 NAS 常驻运行下的内存释放路径，资源导入、订阅、监控和频道同步任务结束后会尝试回收大任务产生的临时内存。
- 订阅日志默认刷新优先读取内存窗口，日志 tail 改为固定长度流式读取，避免高频状态刷新反复整文件解析。
- 订阅分享扫描按“仅目录”和条目预算下传分页限制，减少大分享目录在匹配阶段的一次性拉取量。
- 资源状态和任务状态新增短 TTL 快照层，并在任务创建、更新、清理、资源导入和频道同步后主动失效，降低多端打开时的重复查询压力。
- SSE 增量状态改为轻量监控 / 订阅 payload，图片代理缓存改为增量维护字节数，减少重复 JSON 构建和缓存遍历开销。

- 修复 `/strm/proxy -> /strm/relay` 的 115 `invalid signature`：中继回源统一复用代理解析阶段的同一 User-Agent，避免签名校验因 UA 不一致而失败。
- 优化 `/strm/relay` 令牌策略：默认有效期提升为 2 小时，并在每次访问时自动续期，降低长时间播放后拖动/跳转触发 `410 token 失效` 导致卡顿的问题。
- 优化 `/strm/relay` 跳转稳定性：当上游下载地址过期返回 `401/403/410` 时，自动基于 `pickcode` 刷新下载链接并重试一次，降低长时间播放后 seek 失败概率。
- 优化 `/strm/relay` 播放兼容性：上游 `Range` 错误状态码与关键响应头（如 `Content-Range`）改为透传，避免播放器把可恢复错误误判为 `502` 致闪退。
- 优化 `/strm/proxy` 路径解析性能：新增“目录路径 -> pickcode / CID”内存缓存，减少同目录连续播放、切集时重复目录遍历带来的耗时抖动。
- 优化监控任务生成的 `.strm`：当 115 列表返回 `pick_code` 时写入 `pickcode` 查询参数，播放时可直接跳过路径反查。
- 优化目录树同步性能：多源扫描命中重复文件路径时先去重再写盘，减少重复生成和无效扫描。
